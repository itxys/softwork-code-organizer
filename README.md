# 软著鉴别材料生成器

用于生成程序鉴别材料（源程序）的 DOCX / PDF 输出，符合“前 30 页 + 后 30 页、每页 50 行有效代码”的默认规则。

## 使用方法

1. 安装依赖：`npm install`
2. 启动：`npm start`
3. 在界面选择项目目录、填写软件名称与版本号、选择输出目录。
4. 点击“扫描项目”后再导出。

## 默认规则

- 语言：js/ts、python、java、c/c++、c#、go、rust、php、swift、kotlin
- 排除目录：`node_modules`、`dist`、`build`、`out`、`.git`、`vendor`、`bin`、`obj` 等
- 导出范围：总页数 > 60 时导出前 30 页 + 后 30 页；否则导出全量
- 页眉：`软件名称 + 软件/游戏软件 + V版本号`
- 页脚：页码（“第 X 页”）
- 行号：使用 Word 行号（连续编号）

## 说明

- “有效代码行” = 非空行 + 非注释行；导出时仅保留有效代码行，避免空白行占位。
- 代码抽样按文件路径排序后连续分页，不区分文件边界；长行会按固定列宽换行并计入行号。
- 若可用代码行不足 3000，会在前 30 页区域循环补足，保证第 3000 行仍为某个文件的结尾行。
- PDF 导出依赖本机安装的 Microsoft Word（用于保持 Word 行号效果）。
- 文档鉴别材料模块需要单独的规范或样例支持，当前版本仅覆盖程序鉴别材料。
